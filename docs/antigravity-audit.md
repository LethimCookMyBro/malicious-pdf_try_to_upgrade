# Antigravity Claim Audit

This note records what is implemented versus what was overstated in the
Antigravity/Gemini summary.

## Implemented

- Level 6 prepends a fake JPEG/JFIF header before the PDF bytes.
- Level 8 attempts empty-password encryption for PDFs that PyPDF2 can parse.
- Level 9 attempts object-stream rewriting for PDFs that PyPDF2 can parse.
- Level 5 wraps JavaScript payloads with `eval(unescape("%XX..."))`.
- Level 7 wraps JavaScript payloads in Acrobat/Reader environment checks.
- `test49_1.pdf`, `test49_2.pdf`, and `test49_3.pdf` cover Acrobat
  fingerprinting APIs: `Collab.isDocReadOnly()`, `app.plugIns`, and
  `app.viewerVersion` / `app.viewerType`.

## Overstated

- Level 8 and 9 are best-effort, not universal. Many hand-written PoC PDFs in
  this project intentionally omit complete `xref` / `startxref` structure, so
  PyPDF2 cannot rewrite every generated file.
- The obfuscation levels do not guarantee that Windows Security or another
  antivirus product will stop detecting the files. Defender has detected
  multiple generated PDFs, not only the EICAR test.
- `test11.pdf` intentionally contains the EICAR antivirus test string. It is
  now opt-in via `--include-eicar` instead of being generated by default.

## Remaining Work

- Full xref repair before level 8/9 rewriting.
- Additional JavaScript storage techniques such as annotation metadata
  (`getAnnots()`), trailer `/Info` fields, AcroForm field values, Names tree
  split execution, and incremental-update payloads.
