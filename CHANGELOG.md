# Changelog

All notable changes to this project are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- Three Acrobat JavaScript fingerprinting generators: `test49_1.pdf`
  (`Collab.isDocReadOnly()`), `test49_2.pdf` (`app.plugIns`), and
  `test49_3.pdf` (`app.viewerVersion` / `app.viewerType`).
- Obfuscation levels 5, 7, 8, and 9 in the CLI help: JS `unescape()`
  wrapping, Acrobat/Reader anti-emulation checks, and best-effort
  empty-password encryption/object stream rewriting for PDFs that PyPDF2 can
  parse.
- `PyPDF2` dependency for PDF encryption/object rewrite obfuscation levels.
- `--av-report` mode to list selected files with local Windows
  Security-observed status without writing payload files.
- `--av-report-json PATH` to write the same safety report as a JSON artifact.
- `--exclude-defender-observed` to skip files observed in this workspace's
  Windows Security history.

### Changed
- Profile filtering now actually filters generated tests, and `stealth`
  automatically applies at least obfuscation level 3.
- `test11.pdf` (EICAR antivirus test string) is opt-in via `--include-eicar`
  instead of being generated by default.
- `--list-tests` dry-run mode for inspecting the selected catalog without
  writing payload files.
- Unit tests for catalog counts, EICAR opt-in behavior, profile filtering, and
  Defender-observed safety filtering.
- Smoke-test expected output count now covers the default 71 PDF files plus
  one SVG.
- Split the monolithic script into `malicious_pdf` modules for generators,
  obfuscation, credit injection, and shared URL helpers.
- `docs/antigravity-audit.md` documenting which obfuscation claims are
  implemented versus best-effort or overstated.

## [1.0.1] - 2026-04-20

### Added
- `__version__` constant in `malicious-pdf.py` and a `--version` CLI flag
  so users running from a clone can identify the release they have.
- `CONTRIBUTING.md` describing the contribution workflow and testing policy.
- `CHANGELOG.md` (this file).
- CI smoke-test workflow that runs the generator and asserts the expected
  number of PDF files is produced on every push and pull request.

### Changed
- `CLAUDE.md` documents the release process and the smoke-test count that
  must be bumped when generators are added or removed.

## [1.0.0] - 2026-04-20

First tagged release. Captures the state of the project at the time the
changelog was introduced.

### Added
- 67 PDF generators covering phone-home callbacks, SSRF, XSS, XXE, NTLM
  credential theft, and data exfiltration techniques.
- `--obfuscate` flag with levels 0–4, including base64 JS payload staging
  (level 4) inspired by the April 2026 Adobe Reader 0-day analysis.
- `--output-dir` and `--no-credit` flags.
- Visible credit footer rendered on each generated PDF page.
- Filename + short payload description printed during generation.
- Test cases from 2025–2026 CVEs targeting server-side processors
  (Apache Tika, LibreOffice, Foxit, Apryse).
- CodeQL and Semgrep static-analysis workflows.
- Dependabot configuration.
- `SECURITY.md` with PGP-encrypted private vulnerability reporting.
